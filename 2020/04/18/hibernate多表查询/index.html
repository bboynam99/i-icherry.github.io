Hibernate多表查询

  1. 关于数据库多表查询
    原生SQL
    交叉连接-笛卡尔积(避免)
    select * from A,B
    内连接
    |-隐式内连接
    select * from A,B  where b.aid = a.id
|-显式内连接
    select * from A inner join B on b.aid = a.id
    外连接
    |- 左外
    select * from A left [outer] join B on b.aid = a.id
|- 右外
    select * from A right [outer] join B on b.aid = a.id
  1. HQL多表查询（原则与原生SQL基本一致）
    1. HQL 内连接 => 将连接的两端对象分别返回.放到数组中.
      1. HQL 迫切内连接 => 帮我们进行封装.返回值就是一个对象
      2. HQL 左外连接 => 将连接的两端对象分别返回.放到数组中.
      3. HQL 右外连接 => 将连接的两端对象分别返回.放到数组中.
  2. QBC离线查询
  3. 关于查询优化
    1. 类级别查询(加在class标签上)
      1. get方法:没有任何策略.调用即立即查询数据库加载数据.
      2. load方法: 应用类级别的加载策略
    2. 关联级别查询（夹在set标签）
